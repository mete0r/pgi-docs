Details¶

class Soup.Buffer¶

A data buffer, generally used to represent a chunk of a Soup.MessageBody.

data is a str because that’s generally convenient; in some situations you may need to cast it to #guchar or another type.

static new(data)¶
Parameters:data ([int]) – data
Returns:the new Soup.Buffer.
Return type:Soup.Buffer

Creates a new Soup.Buffer containing length bytes from data.

static new_with_owner(data, length, owner, owner_dnotify)¶
Parameters:
  • data (object) – data
  • length (int) – length of data
  • owner (object) – pointer to an object that owns data
  • owner_dnotify (GLib.DestroyNotify or None) – a function to free/unref owner when the buffer is freed
Returns:

the new Soup.Buffer.
Return type:

Soup.Buffer

Creates a new Soup.Buffer containing length bytes from data. When the Soup.Buffer is freed, it will call owner_dnotify, passing owner to it. You must ensure that data will remain valid until owner_dnotify is called.

For example, you could use this to create a buffer containing data returned from libxml without needing to do an extra copy:

xmlDocDumpMemory (doc, &amp;xmlbody, &amp;len);
return soup_buffer_new_with_owner (xmlbody, len, xmlbody,
                                   (GDestroyNotify)xmlFree);

In this example, data and owner are the same, but in other cases they would be different (eg, owner would be a object, and data would be a pointer to one of the object’s fields).

copy()¶
Returns:the new (or newly-reffed) buffer
Return type:Soup.Buffer

Makes a copy of buffer. In reality, Soup.Buffer is a refcounted type, and calling Soup.Buffer.copy () will normally just increment the refcount on buffer and return it. However, if buffer was created with Soup.MemoryUse.TEMPORARY memory, then Soup.Buffer.copy () will actually return a copy of it, so that the data in the copy will remain valid after the temporary buffer is freed.

free()¶

Frees buffer. (In reality, as described in the documentation for Soup.Buffer.copy (), this is actually an “unref” operation, and may or may not actually free buffer.)

get_as_bytes()¶
Returns:a new GLib.Bytes which has the same content as the Soup.Buffer.
Return type:GLib.Bytes

Creates a GLib.Bytes pointing to the same memory as buffer. The GLib.Bytes will hold a reference on buffer to ensure that it is not freed while the GLib.Bytes is still valid.

get_data()¶
Return type:data: [int]

This function exists for use by language bindings, because it’s not currently possible to get the right effect by annotating the fields of Soup.Buffer.

get_owner()¶
Returns:the owner pointer
Return type:int

Gets the “owner” object for a buffer created with Soup.Buffer.new_with_owner ().

new_subbuffer(offset, length)¶
Parameters:
  • offset (int) – offset within parent to start at
  • length (int) – number of bytes to copy from parent
Returns:

the new Soup.Buffer.
Return type:

Soup.Buffer

Creates a new Soup.Buffer containing length bytes “copied” from parent starting at offset. (Normally this will not actually copy any data, but will instead simply reference the same data as parent does.)
