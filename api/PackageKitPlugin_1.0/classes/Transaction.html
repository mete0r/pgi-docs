set_state(state)¶
Parameters:state (PackageKitPlugin.TransactionState) –
Return type:bool

A transaction can have only one state at any time as it is processed. Typically, these states will be:

  1. ‘new’
  2. ‘waiting for auth’ <— waiting for PolicyKit (optional)
  3. ‘committed’ <— when the client sets the role
  4. ‘ready’ <— when the transaction is ready to be run
  5. ‘running’ <— where PackageKitPlugin.Backend gets used
  6. ‘finished’
