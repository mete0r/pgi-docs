NetworkManager.utils_deinit()¶

Frees all resources used internally by libnm-util. This function is called from an atexit() handler, set up by NetworkManager.utils_init (), but is safe to be called more than once. Subsequent calls have no effect until NetworkManager.utils_init () is called again.

NetworkManager.utils_init()¶
Raises:GLib.GError
Returns:True if the initialization was successful, False on failure.
Return type:bool

Initializes libnm-util; should be called when starting and program that uses libnm-util. Sets up an atexit() handler to ensure de-initialization is performed, but calling NetworkManager.utils_deinit () to explicitly deinitialize libnm-util can also be done. This function can be called more than once.

NetworkManager.utils_security_valid(type, wifi_caps, have_ap, adhoc, ap_flags, ap_wpa, ap_rsn)¶
Parameters:
  • type (NetworkManager.UtilsSecurityType) – the security type to check AP flags and device capabilties against, e.g. NetworkManager.UtilsSecurityType.STATIC_WEP
  • wifi_caps (NetworkManager.DeviceWifiCapabilities) – bitfield of the capabilities of the specific WiFi device, e.g. NetworkManager.DeviceWifiCapabilities.CIPHER_WEP40
  • have_ap (bool) – whether the ap_flags, ap_wpa, and ap_rsn arguments are valid
  • adhoc (bool) – whether the capabilities being tested are from an Ad-Hoc AP (IBSS)
  • ap_flags (NetworkManager.80211ApFlags) – bitfield of AP capabilities, e.g. NetworkManager._80211ApFlags.PRIVACY
  • ap_wpa (NetworkManager.80211ApSecurityFlags) – bitfield of AP capabilties derived from the AP’s WPA beacon, e.g. (NetworkManager._80211ApSecurityFlags.PAIR_TKIP | NetworkManager._80211ApSecurityFlags.KEY_MGMT_PSK )
  • ap_rsn (NetworkManager.80211ApSecurityFlags) – bitfield of AP capabilties derived from the AP’s RSN/WPA2 beacon, e.g. (NetworkManager._80211ApSecurityFlags.PAIR_CCMP | NetworkManager._80211ApSecurityFlags.PAIR_TKIP )
Returns:

True if the device capabilities and AP capabilties intersect and are compatible with the desired type, False if they are not
Return type:

bool

Given a set of device capabilities, and a desired security type to check against, determines whether the combination of device, desired security type, and AP capabilities intersect.

NOTE: this function cannot handle checking security for AP/Hotspot mode; use NetworkManager.utils_ap_mode_security_valid () instead.
