Bases: GObject.GInterface

An object which holds a certificate.

static compare(first, other)¶
Parameters:
  • first (Gcr.Comparable or None) – the certificate to compare
  • other (Gcr.Comparable or None) – the certificate to compare against
Returns:

zero if the certificates match, non-zero otherwise.
Return type:

int

Compare one certificate against another. If the certificates are equal then zero is returned. If one certificate is None or not a certificate, then a non-zero value is returned.

The return value is useful in a stable sort, but has no user logical meaning.

get_basic_constraints(is_ca, path_len)¶
Parameters:
  • is_ca (bool or None) – location to place a True if is an authority
  • path_len (int or None) – location to place the max path length
Returns:

whether basic constraints are present or not
Return type:

bool

Get the basic constraints for the certificate if present. If False is returned then no basic constraints are present and the is_ca and path_len arguments are not changed.

get_fingerprint(type)¶
Parameters:type (GLib.ChecksumType) – the type of algorithm for the fingerprint.
Returns:the raw binary fingerprint
Return type:[int]

Calculate the fingerprint for this certificate.

You can pass GLib.ChecksumType.SHA1 or GLib.ChecksumType.MD5 as the type parameter.

The caller should free the returned data using GLib.free () when it is no longer required.

get_issuer_part(part)¶
Parameters:part (str) – a DN type string or OID.
Returns:the allocated part of the issuer DN, or None if no such part is present
Return type:str

Get a part of the DN of the issuer of this certificate.

Examples of a part might be the ‘OU’ (organizational unit) or the ‘CN’ (common name). Only the value of that part of the DN is returned.

The string returned should be freed by the caller when no longer required.

is_issuer(issuer)¶
Parameters:issuer (Gcr.Certificate) – a possible issuer Gcr.Certificate
Returns:whether issuer could be the issuer of the certificate.
Return type:bool

Check if issuer could be the issuer of this certificate. This is done by comparing the relevant subject and issuer fields. No signature check is done. Proper verification of certificates must be done via a crypto library.

mixin_emit_notify()¶

Implementers of the Gcr.Certificate mixin should call this function to notify when the certificate has changed to emit notifications on the various properties.
