Details¶

class Pango.Matrix¶

A structure specifying a transformation between user-space coordinates and device coordinates. The transformation is given by

x_device = x_user * matrix->xx + y_user * matrix->xy + matrix->x0;
y_device = x_user * matrix->yx + y_user * matrix->yy + matrix->y0;
concat(new_matrix)¶
Parameters:new_matrix (Pango.Matrix) – a Pango.Matrix

Changes the transformation represented by matrix to be the transformation given by first applying transformation given by new_matrix then applying the original transformation.

transform_distance(dx, dy)¶
Parameters:
  • dx (float) – in/out X component of a distance vector
  • dy (float) – in/out Y component of a distance vector
Return type:

dx: float, dy: float

Transforms the distance vector (dx,`dy` ) by matrix. This is similar to Pango.Matrix.transform_point () except that the translation components of the transformation are ignored. The calculation of the returned vector is as follows:

dx2 = dx1 * xx + dy1 * xy;
dy2 = dx1 * yx + dy1 * yy;

Affine transformations are position invariant, so the same vector always transforms to the same vector. If (x1,`y1` ) transforms to (x2,`y2` ) then (x1 +@dx1,@y1+@dy1) will transform to (x1 +@dx2,@y1+@dy2) for all values of x1 and x2.

transform_pixel_rectangle(rect)¶
Parameters:rect (Pango.Rectangle or None) – in/out bounding box in device units, or None
Return type:rect: Pango.Rectangle

First transforms the rect using matrix, then calculates the bounding box of the transformed rectangle. The rectangle should be in device units (pixels).

This function is useful for example when you want to draw a rotated Pango.Layout to an image buffer, and want to know how large the image should be and how much you should shift the layout when rendering.

For better accuracy, you should use Pango.Matrix.transform_rectangle () on original rectangle in Pango units and convert to pixels afterward using Pango.extents_to_pixels ()’s first argument.

transform_rectangle(rect)¶
Parameters:rect (Pango.Rectangle or None) – in/out bounding box in Pango units, or None
Return type:rect: Pango.Rectangle

First transforms rect using matrix, then calculates the bounding box of the transformed rectangle. The rectangle should be in Pango units.

This function is useful for example when you want to draw a rotated Pango.Layout to an image buffer, and want to know how large the image should be and how much you should shift the layout when rendering.

If you have a rectangle in device units (pixels), use Pango.Matrix.transform_pixel_rectangle ().

If you have the rectangle in Pango units and want to convert to transformed pixel bounding box, it is more accurate to transform it first (using this function) and pass the result to Pango.extents_to_pixels (), first argument, for an inclusive rounded rectangle. However, there are valid reasons that you may want to convert to pixels first and then transform, for example when the transformed coordinates may overflow in Pango units (large matrix translation for example).
